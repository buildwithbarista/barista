Vendored Protocol Buffer revisions
==================================

The `.proto` files under this directory are vendored verbatim from upstream
projects and compiled locally at build time by `build.rs` (via
`tonic-prost-build`). They are NOT edited after fetching; their Apache-2.0
license headers are preserved as the attribution chain.

To bump a pin: change the commit hash(es) below, re-fetch the affected files
from the URLs given, and re-run `cargo build -p roastery` so the build script
regenerates the bindings into `$OUT_DIR`.

--------------------------------------------------------------------------------
Repository: https://github.com/googleapis/googleapis
Commit:     c0fcb35628690e9eb15dcefae41c651c67cd050b
Date:       2026-01-31
License:    Apache-2.0

Pin rationale: this is the exact googleapis revision that the REAPI commit
above builds against. The REAPI `MODULE.bazel` at becdd8f declares
`bazel_dep(name = "googleapis", version = "0.0.0-20260130-c0fcb356")` — the
Bazel Central Registry encodes the upstream commit as the trailing short SHA
`c0fcb356`, which resolves to the full SHA above. Pinning to the REAPI's own
googleapis means the vendored `.proto` set is internally consistent: the
google/* messages the REAPI protos import are exactly the ones upstream
compiles them with.

Compilation notes
--------------------------------------------------------------------------------
- `remote_execution.proto` defines all REAPI services (Execution, ActionCache,
  ContentAddressableStorage, Capabilities) in one file. The whole file is
  compiled, but only ContentAddressableStorage + Capabilities are implemented
  by this server (plus ByteStream for large blobs). Execution and ActionCache
  are out of scope for v0.1.
- `google/longrunning/operations.proto` is pulled in only because the REAPI
  Execution service references it; the CAS/Capabilities surface does not use
  long-running operations. It is vendored so the single-file compile of
  `remote_execution.proto` resolves all imports.
- The include root for the proto compiler is this directory (`proto/`). The
  `google/protobuf/*` well-known types resolve from the `protoc` install's
  bundled include path; they are never vendored here.
